## Context

You are working inside a clone of `israel-rent-data-collector`. A second repo, `israel-rent-data`, is cloned **at the same directory level**:

```
adanim_normative_rent/
├── israel-rent-data-collector/   ← you are here
└── israel-rent-data/             ← publication destination
```

`israel-rent-data` is a **data-only** repo. It stores the CSVs generated by this tool so that downstream consumers can access them without running the collector. Your job is to run the pipeline, validate the output, and publish the results there.

## Step-by-Step Tasks

### Step 1 — Install and verify

```bash
cd israel-rent-data-collector
pip install -e ".[dev]"
python scripts/collect.py --help
```

Expected: CLI help text prints without errors.

---

### Step 2 — Probe all endpoints

```bash
python scripts/collect.py --probe
```

This tests connectivity to every data source. Review the output:

- **nadlan**: if `ok: false`, open `nadlan.gov.il` in a browser, open DevTools → Network → XHR, navigate to a city page, and find the actual API endpoint. Update `config.NADLAN_RENT_ENDPOINTS` accordingly.
- **cbs_table49**: if `ok: false`, browse to `https://www.cbs.gov.il/en/subjects/Pages/Average-Monthly-Prices-of-Rent.aspx` and find the current Excel URL. Update `CBS_TABLE49_LATEST_YEAR`, `CBS_TABLE49_LATEST_MONTH`, `CBS_TABLE49_LATEST_LETTER` in `.env` or `config.py`.
- **cbs_api**: if `ok: false`, the CBS API may be temporarily unavailable; skip for now.
- **boi_hedonic**: should always be `ok: true` (it's a formula, no network call).

---

### Step 3 — Discover the live nadlan API shape

After finding a working nadlan endpoint, fetch one locality and inspect the raw response:

```python
from rent_collector.collectors.nadlan import NadlanCollector
import json
c = NadlanCollector()
raw = c._raw_fetch("5000")   # Tel Aviv
print(json.dumps(raw, indent=2, ensure_ascii=False)[:3000])
```

Identify the response shape (A, B, or C — see `01_system_architecture.md`). If the actual shape doesn't match any of the three handled cases, update `_parse_response()` in `collectors/nadlan.py`.

---

### Step 4 — Find the CBS Table 4.9 series in the CBS API (optional)

```bash
python scripts/collect.py --source cbs-api --scan-catalog 2>&1 | head -80
```

Look for a series named something like "Average Monthly Prices of Rent" with a city × room breakdown. If found, add its ID to `CBS_RENT_SERIES` in `config.py`. If not found, skip — CBS Table 4.9 (Step 2) covers this.

---

### Step 5 — Extract BoI hedonic coefficients (already done)

`COEFFICIENTS_ARE_PLACEHOLDER = False` in `boi_hedonic.py` — the coefficients are already set from Appendix B, Table 8 of the BoI paper. No action required unless you want to verify by re-reading the paper at `BOI_HEDONIC_PAPER_URL`.

---

### Step 6 — Run a dry run

```bash
python scripts/collect.py --source all --dry-run
```

All collectors should initialise and log what they would fetch, without making real requests. Fix any import or configuration errors before proceeding.

---

### Step 7 — Run the full pipeline

```bash
python scripts/collect.py --source all --output data/output/rent_benchmarks.csv --validate
```

This may take 30–60 minutes due to polite rate limiting across ~1,200 localities.

**Success criteria:**
- Exit code 0
- `data/output/rent_benchmarks.csv` exists with > 5,000 rows
- Validation passes (total annual normative rent ≥ 131,000,000 NIS)
- Tel Aviv 3-room `rent_nis` is in the range 6,000–9,000
- No rows with `rent_nis < 500` or `rent_nis > 20,000`

If validation fails, check:
1. How many localities have `source = "boi_hedonic"` — if > 500, the nadlan/CBS sources likely failed
2. Run `--source nadlan` alone to isolate the problem
3. Inspect the `notes` column for clues

Also generate the crosswalk:

```bash
python -c "
from rent_collector.utils.locality_crosswalk import get_crosswalk
import pandas as pd
cw = get_crosswalk()
rows = [{'locality_code': l.code, 'locality_name_he': l.name_he, 'locality_name_en': l.name_en} for l in cw.all_localities()]
pd.DataFrame(rows).to_csv('data/output/locality_crosswalk.csv', index=False, encoding='utf-8-sig')
print(f'Crosswalk: {len(rows)} localities')
"
```

---

### Step 8 — Publish to `israel-rent-data` ← NEW

After Step 7 succeeds and validation passes:

```bash
# Copy the validated output to the publication repo
cp data/output/rent_benchmarks.csv ../israel-rent-data/rent_benchmarks.csv
cp data/output/locality_crosswalk.csv ../israel-rent-data/locality_crosswalk.csv

# Commit and push
cd ../israel-rent-data
git add rent_benchmarks.csv locality_crosswalk.csv
git commit -m "data: collection run $(date +%Y-%m-%d)

Sources used: <list which sources yielded rows>
Localities covered: $(python3 -c "import pandas as pd; df=pd.read_csv('rent_benchmarks.csv'); print(df['locality_code'].nunique())")
Validation: PASS"
git push origin main

cd ../israel-rent-data-collector
```

**Do not push to `israel-rent-data` if validation fails.** If validation is marginal (e.g., just under 131M due to known gaps), document the discrepancy in the commit message and flag it for human review.
